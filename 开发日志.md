### v3.0.0 — 移动端重构 + 语音 + GitHub备份

**发布日期：** 2025年1月

#### 新功能

**📱 移动端 / PDA 全面优化**
- 添加 `maximum-scale=1.0, user-scalable=no` 禁止意外双击缩放
- 添加 `apple-mobile-web-app-capable` 支持 iOS 全屏 Web App 模式
- 输入框和按钮高度统一为 54px，符合移动端触摸操作规范
- 扫描确认按钮改为大面积方块（54×54px），单手拇指可轻松触达
- 布局最大宽度限制为 640px，手机竖屏最佳，横屏和平板也正常显示
- 去除所有 `hover` 样式在移动端的副作用，改为 `active` 反馈

**🔊 语音播报功能**
- 基于浏览器原生 `Web Speech API`，无需额外依赖和网络请求
- 扫描后自动朗读篮子编号数字
- 中文模式：逐位读法（如 `12` → "一 二"），仓库环境更清晰不易混淆
- 西班牙语模式：自然数字词（如 `12` → "doce"，`25` → "veinticinco"）
- 数字超过 99 时自动切换为逐位读法，保证清晰度
- 优先选取设备上对应语言的原生语音包
- 语音设置持久化，刷新后保留

**🌐 语言选择**
- 顶部设置栏分段按钮：关 / 中文 / 西语
- 切换语言时自动播放测试音（朗读"1"）
- 通过 `speechSynthesis.onvoiceschanged` 事件预加载语音包列表

**💾 GitHub 备份**
- 使用 GitHub REST API（`PUT /repos/{owner}/{repo}/contents/{path}`）
- 备份路径固定为 `backup/backup_YYYY-MM-DD_HH-MM.json`
- 内容使用 `encodeURIComponent` + `btoa` 正确处理中文 Unicode 编码
- Token 和仓库名持久化到 `localStorage`，下次无需重填
- 上传过程中按钮变为「上传中…」并禁用，防止重复点击
- 完整错误处理：401（Token无效）、404（仓库不存在）、网络异常

#### 改进

- 结果横幅增加 `void el.offsetWidth` 强制 reflow，确保每次扫描动画都重新播放
- 扫描记录上限从 200 条提升到 300 条
- 历史记录 SKU 列增加 `overflow: hidden; text-overflow: ellipsis` 防止长条码撑破布局
- `localStorage` key 升级为 `sortsys_v3`（与旧版数据隔离，避免结构不兼容导致崩溃）
- Toast 提示增加 `max-width: 90vw`，防止长文本在手机上溢出屏幕

### v2.0.0 — 核心功能上线

**发布日期：** 2025年1月（初版）

#### 功能列表

- SKU 条码扫描，自动分配篮子编号（从 1 开始递增）
- 首次出现的 SKU 自动创建新篮子，重复 SKU 指向同一篮子
- 扫描结果大字横幅，绿色（已有）/ 橙色（新建）两种状态
- 篮子格子可视化，扫描时对应格子绿色高亮 3 秒
- 扫描历史记录（最近 200 条），含时间戳、SKU、篮子号
- 手动新增篮子、重置全部功能
- 数据持久化（`localStorage`）
- 响应式布局，支持移动端基本浏览
- 顶部统计：篮子数、总扫描次数、SKU种类数
- 扫描枪支持（Enter 键自动触发）
- 输入框错误时抖动动画

#### 技术架构

- **单文件 SPA**：HTML + CSS + JS 全部内联在一个 `index.html`，便于 GitHub Pages 部署
- **状态管理**：纯 JavaScript 对象 `state`，所有操作通过函数修改后调用 `saveState()`
- **数据结构**：
  ```
  state {
    baskets: [{id, num, sku, count}]   // 篮子列表
    skuMap:  {sku: basketNum}          // SKU → 篮子号 快速查找
    history: [{time, sku, basket, isNew}]  // 扫描历史
    totalScans: number                 // 总扫描次数
  }
  ```
- **字体**：Share Tech Mono（等宽，条码显示）+ Noto Sans SC（中文正文）
- **主题**：深色工业风，CSS Variables 统一色彩管理

## 技术选型说明

### 为什么不用框架（React/Vue）？

| 考量 | 说明 |
|------|------|
| 部署简单 | 单个 HTML 文件，GitHub Pages 直接托管，无需构建步骤 |
| 设备兼容 | 部分仓库 PDA 使用旧版 Android/浏览器，框架兼容性风险更高 |
| 离线可用 | 无外部 JS 依赖（字体除外），网络不稳定环境也能使用 |
| 维护成本 | 仓库操作人员非技术背景，单文件更便于交接和简单修改 |

### 为什么用 localStorage 而不是后端数据库？

- 仓库内网环境可能无外网，纯前端方案最稳定
- 数据量小（百级篮子、千级扫描记录），localStorage 完全够用（5MB 上限）
- 通过 GitHub Backup 功能实现数据持久化和跨设备共享

### Web Speech API 选型

- 浏览器原生 API，无需 SDK，无网络请求，延迟极低（<100ms）
- 仓库场景核心需求：播放快、不卡顿、支持中文
- 缺点：不同设备语音质量有差异，部分 PDA 需要额外安装语音包

## 已知限制

| 问题 | 原因 | 状态 |
|------|------|------|
| 语音在部分 PDA 不工作 | 设备缺少对应语言 TTS 引擎 | 需用户自行安装语音包 |
| 多设备数据不同步 | 纯前端，无后端同步 | 可通过 GitHub 备份手动共享 |
| Safari iOS 首次不播音 | 浏览器安全策略：需用户手势触发 | 首次扫描前先点一下页面 |
| GitHub 备份无法恢复 | 暂未实现从 JSON 导入恢复 | 计划 v4.0 增加 |

## 部署说明

### GitHub Pages 部署步骤

1. Fork 或创建仓库，将 `index.html` 放在根目录
2. 仓库 **Settings** → **Pages**
3. Source 选择 **Deploy from a branch**
4. Branch 选择 `main`，目录选择 `/ (root)`
5. 点 Save，等待约 1 分钟，访问 `https://用户名.github.io/仓库名`

### 目录结构

```
仓库根目录/
├── index.html          # 主程序（全部功能）
├── README.md           # 项目说明
├── 使用教程.md          # 操作手册
├── 开发日志.md          # 本文件
└── backup/             # 自动创建，存放备份JSON
    ├── backup_2025-01-15_09-00.json
    ├── backup_2025-01-15_18-30.json
    └── ...
```
