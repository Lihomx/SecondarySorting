## 1. 系统简介

二次分拣系统用于仓库拣货作业中的 **SKU 二次分拣** 场景：

- 扫描商品条码（SKU），系统自动告知应放入哪个分拣篮
- 每种 SKU 首次扫描时自动创建并分配新篮子编号
- 支持语音播报篮子编号（中文 / 西班牙语）
- 数据保存在本地浏览器，关闭后重新打开仍然保留
- 支持一键备份数据到 GitHub

## 2. 快速开始

### 方式一：直接使用 GitHub Pages

1. 打开部署好的网页地址（例如 `https://你的用户名.github.io/仓库名`）
2. 在手机/PDA浏览器中打开，建议添加到主屏幕书签方便下次访问
3. 开始扫描即可，无需登录、无需安装

### 方式二：本地使用

直接用浏览器打开 `index.html` 文件即可，无需服务器。

| 区域 | 说明 |
|------|------|

## 4. 扫描操作

### 使用扫描枪（推荐）

1. 将光标点击扫描输入框（页面打开时自动聚焦）
2. 用扫描枪扫描商品条码
3. 扫描枪自动发送回车，系统立即响应，**无需手动点确认**
4. 屏幕大字显示篮子号，同时语音播报（如已开启）

### 手动输入

1. 点击输入框，用键盘输入 SKU 编码
2. 按回车键 或 点击 **▶** 按钮 确认

### 结果解读

| 横幅颜色 | 含义 |
|---------|------|
| 🟢 绿色 | 已知 SKU，放入对应篮子 |
| 🟡 橙色 | **新 SKU**，系统已自动创建新篮子并分配编号 |

篮子格子也会同步高亮 3.5 秒，方便快速定位。

## 5. 语音功能

点击设置栏中的语音按钮切换：

| 按钮 | 效果 |
|------|------|
| **关** | 静音，不播报 |
| **中文** | 逐位朗读数字，例如篮子12 → 播报「一 二」 |
| **西语** | 西班牙语数字词，例如篮子12 → 播报「doce」 |

> **说明：** 语音使用浏览器内建的 Web Speech API，无需网络，无需安装。  
> 中文采用逐位读法（1 2 3）而非整数读法（一百二十三），更适合仓库嘈杂环境，不易听错。  
> 首次切换语言时会自动播放「1」作为测试。

### PDA / 安卓注意事项

- 部分 PDA 系统浏览器可能没有对应语言的语音包
- 建议安装 **Google Chrome** 或 **Edge** 浏览器，语音支持更好
- 安卓可在「设置 → 无障碍 → 文字转语音」中安装语音包

## 6. 篮子管理

### 自动创建

每次扫描到 **新 SKU**，系统自动创建下一个编号的篮子，无需手动操作。

### 手动新增

点击「**＋ 新增篮子**」可手动增加一个空篮子（不绑定 SKU），适合预先摆放篮子的场景。

### 篮子格信息

```
┌──────┐
│    5 │  ← 右上角：该 SKU 已扫次数（件数）
│  #3  │  ← 篮子编号
│SKU-A │  ← 绑定的 SKU（截断显示）
└──────┘
```

### 重置

点击「**↺ 重置全部**」清空所有篮子和记录，重新开始新一轮分拣。  
> ⚠️ 重置前建议先备份数据到 GitHub。

## 7. 备份到 GitHub

### 准备工作（只需做一次）

**第一步：创建 GitHub Personal Access Token**

1. 登录 GitHub → 点右上角头像 → **Settings**
2. 左侧菜单滚动到底部 → **Developer settings**
3. **Personal access tokens** → **Tokens (classic)** → **Generate new token**
4. Note 填写备注（如 `仓库分拣备份`）
5. 勾选 **`repo`** 权限（完整仓库读写）
6. 点 **Generate token**，复制并保存好 token（只显示一次！）

**第二步：创建备份仓库**

在 GitHub 新建一个仓库，例如 `warehouse-sort-backup`，可设为私有（Private）。

---

### 执行备份

1. 点击设置栏的「**💾 存 GitHub**」按钮
2. 在弹窗中填写：
   - **Token**：粘贴刚才生成的 `ghp_xxxxx`
   - **仓库名**：格式为 `用户名/仓库名`，例如 `alice/warehouse-sort-backup`
3. 点「**确认上传**」
4. 成功后提示 `✅ 备份成功！backup/backup_2025-01-15_14-30.json`

---

### 备份文件说明

备份保存在仓库的 `backup/` 文件夹，文件名格式：

```
backup_YYYY-MM-DD_HH-MM.json
```

文件内容示例：

```json
{
  "exportedAt": "2025-01-15T06:30:00.000Z",
  "baskets": [
    { "id": 1736921400000, "num": 1, "sku": "SKU-A001", "count": 5 },
    { "id": 1736921450000, "num": 2, "sku": "SKU-B002", "count": 3 }
  ],
  "skuMap": {
    "SKU-A001": 1,
    "SKU-B002": 2
  },
  "history": [
    { "time": "14:23:01", "sku": "SKU-A001", "basket": 1, "isNew": true }
  ],
  "totalScans": 8
}
```

> 🔐 **安全说明：** Token 仅保存在本机浏览器的 `localStorage` 中，不会发送到除 GitHub 以外的任何服务器。建议使用 Fine-grained token 并限制仅访问指定仓库。

## 8. 常见问题

**Q：扫描后没有声音？**  
A：检查手机是否静音；首次使用需手动点击页面（浏览器安全限制，首次交互后才允许播音）；PDA 可在浏览器设置中允许自动播放声音。

**Q：刷新页面后数据丢失？**  
A：数据保存在浏览器 `localStorage`，清除浏览器缓存或使用无痕模式会导致数据丢失。建议定期备份到 GitHub。

**Q：同一台设备多人使用怎么办？**  
A：每次换班前先备份，然后重置，再开始新一轮分拣。

**Q：GitHub 备份失败，提示 404？**  
A：仓库名格式检查：必须是 `用户名/仓库名`（例如 `alice/my-repo`），仓库必须已存在（需要先在 GitHub 上创建）。

**Q：GitHub 备份失败，提示 401？**  
A：Token 过期或权限不足，重新生成一个有 `repo` 权限的 Token。

**Q：多台设备能同步数据吗？**  
A：目前不支持实时同步。可以通过 GitHub 备份 JSON 文件手动共享，后续版本考虑增加从备份恢复功能。

**Q：篮子编号能自定义吗？**  
A：当前版本自动按顺序编号（1、2、3…）。如需自定义，可手动新增篮子后在代码中调整逻辑。
